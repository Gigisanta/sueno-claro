# Autonomous execution plan

## Mission

Build and launch Sueño Claro from docs to working artifact.

## Phase order

1. Web calculator core.
2. Web UI and SEO routes.
3. PWA and analytics/ad abstraction.
4. iOS prototype.
5. RevenueCat premium.
6. Launch QA.

## Execution protocol per phase

1. Read relevant docs and backlog item.
2. Create a short task plan.
3. Implement only the smallest vertical slice.
4. Run tests/build.
5. Use browser QA for UI.
6. Patch issues.
7. Commit with English message.
8. Continue if next task is unblocked.

## Global stop condition

Stop when web MVP is deployed/verified and iOS plan is ready, or when a blocker requires external credentials: Apple Developer account, RevenueCat project/API key, domain/DNS, ad account.

## External credentials needed later

- RevenueCat project public SDK key.
- Apple Developer/App Store Connect access.
- Domain and hosting account.
- Ad provider account if monetizing web.
